# C++ 设计模式

`src` 目录按“基础设计模式分类 + 组合示例”组织，便于分别查看单一模式示例和更贴近业务场景的多模式协作示例。

## 1 创建型（creational）

创建型模式（Creational Pattern）对对象创建过程进行抽象，将“对象如何创建”和“对象如何使用”分离开来，从而降低客户端对具体实现细节的依赖。

### 包含模式

| 模式                          | 应用场景             |
| ----------------------------- | -------------------- |
| 工厂模式（Factory Pattern）   | 根据条件创建不同实例 |
| 单例模式（Singleton Pattern） | 全局唯一对象管理     |
| 原型模式（Prototype Pattern） | 通过复制快速创建对象 |
| 建造者模式（Builder Pattern） | 分步骤构建复杂对象   |

## 2 结构型（structural）

结构型模式（Structural Pattern）描述如何将类或对象组合成更大的结构，从而在保持灵活性的同时构建更复杂的系统。

### 包含模式

| 模式                            | 应用场景                     |
| ------------------------------- | ---------------------------- |
| 代理模式（Proxy Pattern）       | 访问控制、延迟加载、远程代理 |
| 外观模式（Facade Pattern）      | 为复杂子系统提供统一入口     |
| 装饰器模式（Decorator Pattern） | 动态扩展对象功能             |
| 享元模式（Flyweight Pattern）   | 共享细粒度对象以节省内存     |
| 组合模式（Composite Pattern）   | 树形结构的统一处理           |
| 适配器模式（Adapter Pattern）   | 兼容旧接口或第三方接口       |
| 桥接模式（Bridge Pattern）      | 两个变化维度独立扩展         |

## 3 行为型（behavioral）

行为型模式（Behavioral Pattern）关注对象之间的职责划分与交互方式，重点不在对象结构本身，而在系统运行时对象如何协作完成业务。

### 包含模式

| 模式                                          | 应用场景                     |
| --------------------------------------------- | ---------------------------- |
| 模板模式（Template Pattern）                  | 复用稳定流程，定制局部步骤   |
| 策略模式（Strategy Pattern）                  | 支付、排序、压缩等算法切换   |
| 责任链模式（Chain of Responsibility Pattern） | 审批流、过滤链、异常处理链   |
| 迭代器模式（Iterator Pattern）                | 统一遍历集合或容器           |
| 命令模式（Command Pattern）                   | 撤销重做、任务队列、操作日志 |
| 状态模式（State Pattern）                     | 订单状态流转、状态驱动行为   |
| 备忘录模式（Memento Pattern）                 | 草稿、快照、回滚恢复         |
| 中介者模式（Mediator Pattern）                | 聊天室、事件中心、组件协调   |
| 解释器模式（Interpreter Pattern）             | 规则表达式、领域语法解析     |
| 观察者模式（Observer Pattern）                | 事件订阅、消息通知、联动刷新 |
| 访问者模式（Visitor Pattern）                 | 对稳定结构附加多种操作       |
| 委派模式（Delegate Pattern）                  | 任务分发、职责转交           |

## 4 组合示例（combinations）

组合示例用于展示多个设计模式如何协同解决更接近真实业务的问题，例如撤销 / 重做、聊天室分发、文件系统遍历等。

### 当前示例

| 组合            | 典型场景                                     | 组合说明                                             |
| --------------- | -------------------------------------------- | ---------------------------------------------------- |
| 备忘录 + 命令   | 编辑器撤销 / 重做、游戏回退、事务回滚        | `Command` 负责操作流程，`Memento` 负责保存和恢复状态 |
| 策略 + 工厂     | 支付方式切换、压缩算法选择、校验算法动态选择 | `Factory` 负责创建具体策略，`Strategy` 负责算法执行  |
| 组合 + 访问者   | 文件系统遍历、AST 编译处理、控件树导出       | `Composite` 负责树结构，`Visitor` 负责附加操作       |
| 观察者 + 中介者 | UI 组件联动、聊天室消息分发、事件通知中枢    | `Mediator` 负责集中调度，`Observer` 负责广播通知     |
