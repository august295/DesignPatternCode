# 备忘录 + 命令

## 典型场景

- 编辑器 / IDE 的撤销 / 重做
- 游戏的回退上一步操作
- 事务回滚

## 组合方式

- `Command`

  封装编辑行为，统一提供 `Execute()`、`Undo()`、`Redo()`

- `Memento`

  保存受影响区间的位置、旧文本、新文本，不暴露编辑器内部实现细节

## 为什么这样组合

`Command` 负责管理一次操作的执行时机、撤销和重做流程；
`Memento` 负责记录一次变更所需的最小恢复信息，而不是整篇文档快照。

两者结合后，调用者不需要知道编辑器如何保存内部状态，只需要调度命令即可完成撤销 / 重做。

## 这版示例的优化点

每条命令只记录“变更区间”：

- 插入：记录插入位置和新文本
- 删除：记录删除位置和被删旧文本

这样更贴近真实编辑器对撤销栈的实现思路，也避免了每一步都保存整篇内容。
