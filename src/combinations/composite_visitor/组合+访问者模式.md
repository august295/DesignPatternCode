# 组合 + 访问者

## 典型场景

- 文件系统遍历并执行不同操作，如统计大小、搜索、压缩
- AST（抽象语法树）的编译处理
- UI 控件树导出为 XML / JSON

## 组合方式

- `Composite`

  表达树状结构，本示例中使用目录和文件组织文件系统层次

- `Visitor`

  对节点执行不同操作，如统计总大小、搜索文件名，而不把这些逻辑塞进节点类

## 为什么这样组合

组合模式负责稳定的结构表达，访问者模式负责不断变化的操作扩展。

如果把“统计大小”“搜索”“压缩”“导出”等逻辑都直接加到目录和文件节点里，
节点类会随着操作数量增加而不断膨胀。使用访问者后，可以在不修改节点类的情况下增加新操作。
